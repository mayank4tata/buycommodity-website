# BCSPL Website V2.2.0 Deployment Guide

V2.2.0 is a cumulative upgrade over V2.1.4. It retains password recovery and
Users & Access, and adds the unified Catalogue & Pricing Master and Home pricing search.

## 1. Pre-deployment backup

1. Export the current Products Master, Grade Master and Product Pricing from Admin.
2. Take a Supabase database backup/snapshot.
3. Keep the currently deployed website ZIP and Git tag available for rollback.
4. Confirm that the production website is presently stable before starting.

## 2. Confirm V2.1.4 prerequisites

If V2.1.4 was not fully deployed previously, complete these first:

1. Run `supabase/migrations/20260724_v2_1_4_admin_access.sql`.
2. Add `https://buycommodity.in/admin-reset-password.html` to Supabase Auth Redirect URLs.
3. Deploy the `manage-admin-users` Edge Function with its required secrets.
4. Confirm Forgot Password and Users & Access are working.

Do not rerun or replace the service-role key in browser code. Auth Admin operations must
remain inside the Edge Function.

## 3. Confirm ordering prerequisite

V2.2.0 uses the ordering helper functions supplied with the earlier release. Confirm that
`supabase/migrations/20260712_rc13_ord_003_transactional_ordering.sql` has been run.
If uncertain, run it again; it is designed as an idempotent function deployment.

## 4. Run the V2.2.0 database migration

In Supabase SQL Editor, run:

`supabase/migrations/20260724_v2_2_0_catalogue_pricing_master.sql`

This migration:

- Adds `display_order` to `product_categories` and backfills existing categories.
- Creates `catalogue_import_audit` with administrator-only read access.
- Creates the secure transactional RPC `admin_import_catalogue_master_v2(jsonb)`.
- Grants RPC execution to authenticated users; the function itself verifies Admin access.

The migration is wrapped in a transaction. Resolve any SQL error before uploading the
new website files.

## 5. Upload website files

Upload the full contents of `01_Source Code` to the production website repository,
preserving directories and filenames. Do not upload only the changed JavaScript files.

Important files include:

- `admin.html`
- `index.html`
- `enquiry.html`
- `products.html`
- `js/app.js`
- `js/supabase.js`
- `js/v2.2-catalogue-master.js`
- `js/v2.2-home-search.js`
- `css/v2.2-catalogue-master.css`
- `css/v2.2-home-search.css`
- `templates/BCSPL_Catalogue_Pricing_Master_Template_V2.2.0.xlsx`

The obsolete root-level `app.js` is intentionally absent. All pages load
`js/app.js?v=2.2.0`.

## 6. Cache and deployment verification

1. Wait for GitHub Pages/custom-domain deployment to finish.
2. Open the site in an InPrivate/Incognito window.
3. Hard refresh with `Ctrl + Shift + R`.
4. In browser Developer Tools > Network, confirm these return HTTP 200:
   - `js/app.js?v=2.2.0`
   - `js/v2.2-home-search.js?v=2.2.0`
   - `js/v2.2-catalogue-master.js?v=2.2.0`
5. Confirm no old root `/app.js` is being requested.

## 7. V2.1.4 regression checks

Complete these before testing the new master:

- Admin email/password login.
- Forgot Password email request.
- Password reset page and new-password update.
- Users & Access visible only to Super Admin.
- Invite Admin, resend access link and send password-reset email.
- Disable/reactivate user protections.
- Enquiries, Enquiry Recipients, Image Library and Site Settings.

## 8. Catalogue & Pricing Master checks

1. Open Admin > Catalogue & Pricing Master.
2. Select **Download Current Master**.
3. Confirm the workbook contains:
   - Instructions
   - Catalogue_Pricing_Master
   - hidden Reference_Lists
4. Confirm visible columns use Segment, Category, Product, Grade and Price names.
5. Confirm system ID columns are hidden at the far right.
6. Upload the downloaded workbook without changing it.
7. Confirm it opens in the HTML review and shows no database changes.
8. Test a controlled update:
   - change one rate or remark;
   - keep Price Validity Days at 1;
   - retain the required Pricing Display Order;
   - upload, review, Revalidate and Save All Valid Changes.
9. Test a new Grade/Pricing row under an existing Product with hidden IDs blank.
10. Test an invalid date such as `23`; confirm the issue identifies the Product/Grade/Size,
    shows the entered value and explains the exact correction.
11. Test Cancel Row Changes, Remove from Upload and Cancel Complete Upload.
12. Test explicit Delete only on a non-critical test pricing row, then restore it.
13. Confirm an audit reference is returned after a successful save.

## 9. Legacy workbook compatibility check

Upload a copy of the earlier Product Pricing export. The V2.2.0 review should resolve
existing products/grades from hidden IDs or unique Product Name + Category values. It must
not write to the database until **Save All Valid Changes** is confirmed.

## 10. Home pricing search checks

1. Open Home in an Incognito window.
2. Enter `HR` in the search bar.
3. Confirm matching HR products and pricing rows appear below the search bar.
4. Confirm the result shows Grade, Specification/Size and Price/Price on Request.
5. Confirm the search does not redirect to the general Products page.
6. Select **View all rates** and confirm Prices & Enquiry opens with the relevant filter.
7. Click a Home search chip and confirm it performs the same pricing search.

## 11. Rollback

If the static release must be rolled back, redeploy the prior V2.1.4 website package.
The added Category Display Order, audit table and RPC can safely remain unused. Do not drop
or alter production data without a separate reviewed rollback plan.

## 12. Production sign-off

Record:

- Deployment date/time
- Git commit/tag
- Person deploying
- SQL migration result
- V2.1.4 regression result
- Master import test audit reference
- Home search test result
